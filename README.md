# Продвинутый system design

![Обложка продвинутого system design](weeks_of_coding_save_hours_of_designing.png)

Привет! Мы начинаем продвинутую часть курса по системному дизайну.

Меня зовут Михаил, я писал backend на php, сейчас являюсь ментором.

В продакшене почти неизбежно что‑то падает: железо, сеть, отдельные сервисы.  
Но до отказов пользователи в первую очередь сталкиваются с задержками и странным поведением под нагрузкой:
редкие хвосты p99, “плавающий” throughput, внезапные очереди.

Чтобы потом осознанно говорить про отказоустойчивость, нужно сначала договориться,
как мы вообще измеряем и контролируем живую систему: latency, tail latency, throughput,
распределение задержек, что происходит с ними при росте нагрузки.

В этой главе мы начнём именно с этого — с базовой механики производительности и хвостов —
а дальше поверх неё будем разбирать шардирование, распределённые транзакции и продвинутое кэширование:
то есть те инструменты, которыми вы реально будете спасать сервис, когда что‑то пойдёт не так.
