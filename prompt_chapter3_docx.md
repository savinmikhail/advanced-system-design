## Промпт для сборки главы 3 в один DOCX

Контекст: в репозитории `advanced-system-design` лежит сценарий главы 3, разбитый по подпапкам. 
В каждой подпапке есть свой `readme.md` с текстом части, а в корне есть скрипт `md_to_docx.py` и уже настроенное виртуальное окружение `.venv` с `python-docx`.

Задача для ассистента: собрать **один общий DOCX** по всей главе 3 с учётом картинок.

Пожалуйста, сделай следующее:

1. Работай в корне репозитория `advanced-system-design`.
2. Найди все файлы `readme.md` во вложенных папках и упорядочь их:
   - сначала корневой `./README.md`
   - сначала все из `00_performance` (внутри — по подпапкам),
   - затем из `01_sharding`,
   - затем `02_distributed-transactions`,
   - затем `03_cache`,
   - затем `04_trade_offs`,
   - внутри каждой папки — по имени подпапки и файла.
3. Объедини их содержимое в один файл `chapter3_full.md` в корне репозитория:
   - перед каждым куском ставь HTML-комментарий с относительным путём к исходному файлу от корня репозитория, например:  
     `<!-- 00_performance/00_basics/readme.md -->`.
   - между кусками добавляй разделитель:  
     `---` (с пустыми строками до и после).
4. Обязательно **сохрани и поправь картинки**:
   - в исходных `readme.md` картинки записаны как `![alt](relative-path.png)` относительно папки, где лежит этот `readme.md`;
   - в итоговом `chapter3_full.md` нужно заменить путь так, чтобы он был относителен корня репозитория.
   - то есть, если картинка была в `01_sharding/02_dynamic_sharding_strategies/img.png`, а в тексте было `![...](img.png)`, в итоговом файле должно стать:  
     `![...](01_sharding/02_dynamic_sharding_strategies/img.png)`.
5. Для шага 3–4 напиши и запусти небольшой Python-скрипт (можно прямо через `python3 - << 'PY'`), который:
   - обходит все `readme.md` (кроме корневого),
   - создаёт `chapter3_full.md`,
   - для строк, которые **полностью** состоят из markdown-изображения (`![alt](path)`), переписывает путь с учётом родительской папки,
   - остальные строки копирует как есть.
6. После того как `chapter3_full.md` готов, вызови скрипт конвертации в DOCX:
   - используй уже существующее окружение:  
     `./.venv/bin/python md_to_docx.py chapter3_full.md chapter3_full.docx`
   - на выходе должен получиться файл `chapter3_full.docx` в корне репозитория.
7. В конце ответь, что именно ты сделал, и перечисли итоговые файлы: `chapter3_full.md` и `chapter3_full.docx`.

Важно:
- не трогай сами исходные `readme.md`;
- не меняй структуру папок и имена файлов;
- проверь, что картинки в итоговом DOCX подтягиваются (пути в markdown должны указывать на реально существующие файлы в репозитории).
